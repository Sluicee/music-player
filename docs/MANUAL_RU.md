# Инструкция пользователя

Данное руководство объясняет, как программа «Memory Card» распознает вашу музыкальную коллекцию и как лучше организовать файлы для корректного отображения.

## Распознавание файлов

### Поддерживаемые форматы
Программа поддерживает следующие расширения файлов:
- **Аудио:** `.mp3`, `.flac`, `.ogg`, `.m4a`, `.aac`, `.wav`, `.opus`.
- **Обложки:** `.jpg`, `.jpeg`, `.png`.

### Автоматическая группировка
Плеер объединяет треки в альбомы, используя метаданные (теги) внутри файлов:
1. **Альбомы:** Группировка происходит по тегам **«Альбом»** и **«Исполнитель альбома»** (Album Artist). Если «Исполнитель альбома» не указан, используется основной «Исполнитель».
2. **Многодисковые альбомы:** Если в названии альбома указано "(Disc 1)" или "[CD 2]", программа автоматически скроет эти суффиксы для группировки. Весь альбом будет отображаться как одна сущность, а треки внутри будут отсортированы сначала по номеру диска, затем по номеру трека.
3. **Гостевые участия (Feat.):** При определении владельца альбома программа игнорирует приписки вроде `feat.`, `ft.` или `featuring` в имени исполнителя. Это позволяет альбомам артиста не разделяться, если в некоторых треках указаны гости.

### Обложки альбомов
Программа ищет обложку в следующем порядке:
1. **Встроенные теги:** Сначала проверяются изображения, зашитые в сам аудиофайл.
2. **Файлы в папке:** Если встроенной обложки нет, программа ищет в папке с треком файлы с именами: `cover`, `folder`, `front`, `album`, `albumart` или `artwork`.
3. **Любое изображение:** Если ничего из вышеперечисленного не найдено, будет выбрано первое попавшееся изображение из папки.

> [!TIP]
> Для лучшей производительности и качества рекомендуется использовать обложки размером **600x600 пикселей**. Программа автоматически оптимизирует слишком большие изображения при сканировании.

## Рекомендации по организации библиотеки

Для наиболее быстрого и корректного сканирования рекомендуется следующая структура папок:
```text
Музыка/
└── Исполнитель/
    └── [Год] Название альбома/
        ├── cover.jpg (желательно)
        ├── 01 - Трек.flac
        └── 02 - Трек.flac
```
Хотя программа может распознать файлы, даже если они свалены в одну кучу (благодаря тегам), структурированное хранение упрощает поиск и обновление коллекции.

## Управление (Горячие клавиши)

| Действие | Клавиша |
| :--- | :--- |
| **Play / Pause** | `Space` (Пробел) |
| **Предыдущий трек** | `←` (Стрелка влево) |
| **Следующий трек** | `→` (Стрелка вправо) |
| **Громкость выше** | `↑` (Стрелка вверх) |
| **Громкость ниже** | `↓` (Стрелка вниз) |
| **Mute / Unmute** | `M` |
| **Shuffle (Square ⏹)** | `S` (Перемешать библиотеку) |
| **Repeat (Triangle 🔼)** | `R` (Режимы повтора: Один / Все / Нет) |
| **Search (Circle ⏺)** | `F` или `/` (Открыть поиск) |
| **Назад / Закрыть** | `Escape` (Закрыть окна и оверлеи) |
| **Вкладки (Tabs)** | `1` — Библиотека, `2` — Плейлисты |
